## Architecture

**Stack:** React 19 + TypeScript, Vite, MUI v9, TanStack Router v1, TanStack Query v5, React Hook Form + Zod.

**Routing** uses TanStack Router's file-based routing. `src/routeTree.gen.ts` is auto-generated by the `@tanstack/router-plugin` Vite plugin — never edit it manually. Route files live in `src/routes/`:

- `/` → `routes/index.tsx` — login page; redirects to `/home` if token exists
- `/signup` → `routes/signup.tsx`
- `/_authRoutes/` — layout route that guards all authenticated pages; checks `localStorage.getItem("token")` in `beforeLoad` and redirects to `/` if absent
- `/_authRoutes/home` → `routes/_authRoutes/home.tsx`

**Auth flow:** Token is stored in `localStorage` under the key `"token"`. Login calls `POST /api/token/` (Django JWT endpoint), signup calls `POST /users/register/`. Both are wrapped as TanStack Query mutations in `src/hooks/useToken.ts` and `src/hooks/useRegister.ts`. API base is `http://localhost:8000`.

**Feature modules** live in `src/features/<domain>/` with an `index.ts` barrel. Each feature owns its page components, form components, Zod schemas (`schemas.ts`), and TypeScript types (`types.ts`).

**Hooks** in `src/hooks/` contain API mutation hooks. `src/hooks/types.ts` defines shared types (e.g., `ValidationErrors`).

**Backend API error shape:** Errors are returned as `{ [field]: string[] }` (Django REST Framework validation errors). Hooks read `errors[keys[0]][0]` and throw as an `Error` message.

## UI

This project uses Material UI for components. Use the mui-mcp when working with this library

**Responsiviness**

Whe kept the default material ui breakpoints and they should be used for responsiviness. We are only concerned with 4 main breakpoints: phone, tablet and desktop. Heres how those devices fit whithin material ui breakpoints:

xs, extra-small: 0px -> Phone
sm, small: 600px -> Tablet
md, medium: 900px -> Tablet
lg, large: 1200px -> Desktop
xl, extra-large: 1536px -> Desktop
